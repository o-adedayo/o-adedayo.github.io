Detection Engineering

Rules with documentation, not just YAML.

Sigma rules published on GitHub, each with the ATT&CK mapping, the telemetry it depends on, known false-positive sources, and the test procedure used to validate it. 

# sigma/windows/proc_creation_schtasks_persistence.yml
title: Scheduled Task Creation From Scripting Host
status: experimental
description: Detects schtasks.exe spawned by script
  interpreters — a common persistence pattern.
tags:
  - attack.persistence
  - attack.t1053.005
logsource:
  category: process_creation
  product: windows
detection:
  selection:
    Image|endswith: '\schtasks.exe'
    ParentImage|endswith:
      - '\wscript.exe'
      - '\cscript.exe'
      - '\powershell.exe'
  condition: selection
falsepositives:
  - Admin automation scripts (baseline first)
level: medium
